New alpha feedback uncovered another critical archive/upload workflow bug. Please investigate and fix carefully without breaking existing archive, collection, or portfolio logic.

ISSUE:
After adding one photo successfully, users sometimes cannot add additional photos to Archive.
The upload flow appears to become stuck on the last uploaded image.

Symptoms reported:
- “It’s stuck on the last photo I added.”
- “Won’t let me add to archive.”
- Upload UI may freeze or fail silently after first upload.
- Possibly related to upload state not resetting properly.

Likely causes to investigate:
- upload loading state never resets
- stale image URI state
- image picker cache
- failed async promise handling
- upload queue deadlock
- state mutation issue
- optimistic update not resolving
- archive refresh not triggering
- React state dependency issue

Requirements:
1. Users must be able to:
   - upload multiple images consecutively
   - upload from iPhone and Android
   - add multiple archive pieces in one session
   - continue uploading after failed upload attempts

2. Add proper:
   - loading reset handling
   - upload error handling
   - retry handling
   - success/failure feedback
   - state cleanup after upload completes

3. Ensure:
   - image picker fully resets after upload
   - upload button becomes active again
   - previous image state does not block new uploads
   - archive list refreshes correctly after upload

4. Carefully audit:
   - useEffect dependencies
   - async upload flow
   - image upload hooks
   - Supabase upload responses
   - image processing pipeline
   - archive creation state logic

5. Preserve:
   - existing visual design
   - current archive layouts
   - collection logic
   - portfolio behavior
   - hero image behavior

6. Add lightweight debugging logs temporarily for:
   - image selection
   - upload start/end
   - upload failure reason
   - state reset confirmation

This upload flow is now top priority because Archive is the foundation of the entire GlazeVault workflow.
